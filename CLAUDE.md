# CLAUDE.md — 非交渉ルール

このファイルは Claude Code がこのリポジトリで作業する際に必ず従う非交渉ルールを定義する。

## 絶対禁止

- secrets（APIキー・トークン・パスワード）をコードや設定ファイルに直書きしない
- `.env` ファイルをコミットしない（`.gitignore` に必ず含める）
- 公式URLが存在しないデータを投稿対象にしない
- 期限切れのデータを投稿対象にしない
- スコアが最低閾値を下回るデータを投稿対象にしない
- `state/state.json` に記録済みの ID を再投稿しない
- 280文字を超える X 投稿テキストを生成しない
- 同一文率が高いテンプレートを連続使用しない（同じテンプレートを3連続禁止）

## 必須要件

- すべての外部 API 呼び出しにタイムアウトを設定する（デフォルト 15 秒）
- すべての例外をキャッチしてログに記録し、プロセスを継続する
- `logs/` に日付ごとの監査ログを JSON Lines 形式で残す
- `--dry-run` モードではファイル書き込みと X 投稿を一切行わない
- `POST_ENABLED=false` の場合は X 投稿をスキップする（サイト生成は行う）
- 新しいコレクターを追加する際は `tests/` にテストを追加する
- `site/` に書き出すファイルは UTF-8 で出力する

## コード品質

- 型ヒントを使用する（`from __future__ import annotations` を各ファイルの先頭に）
- `logging` モジュールを使用し `print()` での出力は禁止
- 関数は単一責任の原則に従う（1 関数 = 1 処理）
- マジックナンバーは定数として定義する
- `pytest` が `python -m pytest tests/` で全テスト通過すること

## 収益化ルール

- `MONETIZATION_ENABLED` 環境変数が `true` の場合のみ収益リンクを表示する
- 初期値は `false`（コード上のデフォルト）
- 収益リンクは `site/` テンプレートの指定スロットにのみ配置する
- 収益目的でデータのスコアや順位を操作しない

## データソース

- 公式ドメインからのデータのみ受け付ける
- スクレイピングよりも公式 API・RSS を優先する
- 新しいソースを追加する前に `collector.md` エージェントのチェックリストを確認する

## スキル・エージェント

- `.claude/skills/` 配下のスキルを活用してタスクを実行する
- `.claude/agents/` 配下のエージェント定義に従って役割を分担する
